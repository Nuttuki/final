จากภาพจงอธิบายหน้าที่ของ ROS Master 
- ทำหน้าที่เป็น server สำหรับใช้ node หลายๆตัวพร้อมกันและส่งข้อความหากันได้

อธิบายการทำงานของ ROS Topics และ ROS Service
- topics ข้อความจะถูกส่งให้ มีการส่งข้อความของโหนดคือ  publishing โหนดที่สนใจในข้อมูลบางประเภทจะทำการ subscribe หัวข้อที่ต้องการอาจมีการ publish / subscribe หลายรายพร้อมกันสำหรับหัวข้อเดียวและโหนดเดียวอาจ publish / subscribe ได้หลาย Topics โดยแต่ละโหนดมีหน้าที่ต่างกัน
- services รูปแบบการ publish / subscribe เป็นแนวคิดในการสื่อสารที่ยืดหยุ่นมาก แต่การขนส่งทางเดียวอย่างต่อเนื่องนั้นไม่เหมาะสม บางครั้งจึงมีการใช้บริการ Request / reply กำหนดการส่งข้อความ โดยสร้างข้อความหนึ่งสำหรับการร้องขอและอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไปยังโหนดที่ร้องขอ

อธิบายความแตกต่างของ Topic กับ message
- topic นั้นจะเป็นแบบ asynchronousคือทำหน้าที่แบบไม่สนสิ่งอื่น อยากส่งก็ส่ง อยากรับก็รับ
- การส่ง Message ระหว่าง Node ต่างๆใน ROS. การส่งจะเป็นแบบ Stream ข้อมูลแบบต่อเนื่อง ส่งไป มีคนรับไม่รับไม่รู้ คนส่งมีหน้าที่ส่ง อยากจะส่งเมื่อไร ก็ส่ง คนรับ ก็รอรับอย่างเดียว

อธิบายความแตกต่างในเชิงการใช้งานของ ROS Topic กับ ROS Service
- ros topic ใช้เพื่อดูว่าในเเต่ละ topic มีรายละเอียดเป็นยังไงบ้าง มันจะแสดงว่า topic นี้ ส่งข้อมูลเป็นประเภทอะไร หรือรับข้อมูลเป็นประเภทอะไร สามารถดูได้ว่า ส่งค่าหรือรับค่า ถ้าส่งค่า จะบอกว่าส่งค่า(Publishers)ไปไหนที่ node ไหน หรือรับค่า(Subscribers)มาจาก node ไหน
-ros services สามารถกำหนดการ Request / reply ในการส่งข้อมูล  การส่ง Message แบบ ไม่เน้นความต่อเนื่อง มี Request มี Response
